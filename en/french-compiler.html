Project Overview

This project implements a complete compiler for a custom programming language inspired by French keywords.

The compiler reads source code, builds structured representations, validates semantic rules and generates compilable Java code.

The project is organized as a real compilation pipeline including:

  • Lexical analysis
  • Parsing
  • AST construction
  • Semantic analysis
  • Intermediate Representation (IR)
  • Optimizations
  • Java code generation

Technical Components

Lexical Analysis

The lexer converts source text into tokens and keeps position information for precise diagnostics.

AST

The abstract syntax tree represents the logical structure of the program: functions, blocks, statements and expressions.

Intermediate Representation

The IR makes the compiler more modular and prepares inspection, optimization and code generation steps.

Diagnostics

The diagnostics system centralizes errors with lines, columns, intervals and messages to support debugging.

AST → IR → Java Conversion

The project separates responsibilities clearly: the AST describes the source program, the IR simplifies internal transformations, and the generator produces compilable Java code.

  • Structured conversion with AstVersIr.
  • Java generation through IrVersJava and JavaGenerator.
  • Runtime support for functions such as lire().
  • Maven/JUnit tests to validate the IR model and generated code.
