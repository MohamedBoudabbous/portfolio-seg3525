eHotels Management System

Hotel management web interface designed with a strong focus on navigation, user flows, forms, readability and usability.

UI Design Spring Boot PostgreSQL User Flows CRUD Interfaces

Project Overview

eHotels is a complete hotel management interface designed to manage hotel chains, hotels, rooms, clients, employees, reservations and rentals.

The main objective of the project was to design a clear, coherent and user-friendly interface capable of simplifying complex administrative operations.

UI/UX Design Goals

Clear Navigation

Main sections are accessible through a persistent navigation bar: search, reservations, rentals, clients, employees, hotels and rooms.

Guided Forms

Forms are organized into logical groups in order to reduce cognitive load and simplify data entry.

Readable Data

Tables display essential information with visual status indicators, action buttons and a consistent structure.

Information Architecture and Modeling

ER Diagram

The ER diagram was used to structure the main entities and understand the relationships required before designing the interfaces.

Mermaid Representation

This representation helped transition from conceptual modeling to relational logic and interface organization.

User Flow: Search and Reservation

Multi-Criteria Search

The search interface allows filtering rooms by dates, capacity, size, hotel chain, category, price and location.

Reservation Creation

The reservation form guides users through client selection, room selection, dates and reservation status.

Reservation and Rental Management

Reservation List

Reservations are displayed with statuses and conversion actions whenever a reservation can be transformed into a rental.

Rental Conversion

The conversion flow transforms a confirmed reservation into an active rental with an assigned employee.

Direct Rental

Users can create a rental without a prior reservation through a dedicated interface.

Rental Tracking

Active and completed rentals are displayed with essential operational information.

Administrative Interfaces

Client Management

The client interface displays profiles, statuses, modification actions and archive actions.

Client Creation

The client form groups together personal information, social insurance number, registration date and status.

Employee Management

Employees are displayed with their role, associated hotel, location and status.

Employee Creation

The employee form allows administrators to create employees and assign them to hotels and roles.

Hotels and Rooms

Hotel Creation

The hotel form allows administrators to define the chain, category, address, location and status.

Room Management

Rooms are displayed with pricing, capacity, size, view, additional bed and condition.

Room Creation

The room creation form allows new rooms to be attached to existing hotels.

Applied Interface Design Principles

Visual Consistency

All pages share a consistent structure with headers, forms, tables and action buttons.

Action Visibility

Main actions such as add, edit, archive, convert and return are clearly visible.

Complexity Reduction

Complex operations are separated into specialized interfaces in order to simplify the user experience.
