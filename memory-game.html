CardGame - Jeu de mémoire

Jeu de mémoire modulaire en C++ conçu avec la programmation orientée objet, plusieurs modes de jeu, des tests automatisés et un système de build avec CMake.

C++ POO CMake Tests

Présentation du projet

CardGame est un jeu de mémoire en console développé en C++. Le projet a été conçu pour pratiquer la programmation modulaire, la conception orientée objet et la logique d’un jeu interactif.

Le code est organisé autour de responsabilités distinctes : logique du jeu, interface utilisateur, gestion des règles et tests automatisés.

Progression du gameplay

Menu principal

Le jeu démarre avec plusieurs modes disponibles, notamment le mode normal, le mode expert et les tests automatisés.

Plateau de jeu

Le joueur interagit avec un plateau de cartes cachées qui doivent être révélées et associées correctement.

Système de correspondance

Le jeu valide les cartes sélectionnées et met à jour le score lorsqu’une correspondance est détectée.

Règles avancées et tests

Règles expertes

Le mode expert ajoute une logique de jeu plus avancée et des règles supplémentaires.

Gestion des tours

Le projet gère les tours des joueurs et les actions attendues pendant le déroulement du jeu.

Tests automatisés

Les tests valident les comportements essentiels comme la logique des cartes, la génération du plateau et la gestion des règles.

Conception technique

Structure du projet

Le projet est séparé en modules pour améliorer la lisibilité, la maintenabilité et l’organisation.

Conception POO

L’implémentation utilise des classes pour modéliser les cartes, le plateau, les joueurs et les règles.

Système de build

Le projet utilise CMake pour organiser la compilation et générer l’exécutable.
