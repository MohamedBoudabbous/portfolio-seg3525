Présentation du projet

Ce projet consiste à construire un compilateur complet pour un langage de programmation personnalisé utilisant des mots-clés inspirés du français.

Le compilateur lit un programme source, construit des représentations structurées, valide les règles sémantiques et génère du code Java compilable.

Le projet est organisé comme un véritable pipeline de compilation incluant :

  • Analyse lexicale (Lexeur)
  • Analyse syntaxique (Parseur)
  • Construction de l’AST
  • Analyse sémantique
  • Représentation intermédiaire (IR)
  • Optimisations
  • Génération de code Java

Composants techniques

Analyse lexicale

Le lexeur transforme le texte source en jetons et conserve les informations de position afin de produire des diagnostics précis.

AST

L’arbre syntaxique abstrait représente la structure logique du programme : fonctions, blocs, instructions et expressions.

Représentation intermédiaire

L’IR rend le compilateur plus modulaire et prépare les étapes d’inspection, d’optimisation et de génération de code.

Diagnostics

Le système de diagnostics centralise les erreurs avec lignes, colonnes, intervalles et messages pour faciliter le débogage.

Conversion AST → IR → Java

Le projet sépare clairement les responsabilités : l’AST décrit le programme source, l’IR simplifie la manipulation interne, puis le générateur produit du Java compilable.

  • Conversion structurée avec AstVersIr.
  • Génération Java avec IrVersJava et JavaGenerator.
  • Support de fonctions comme lire() avec injection du runtime Java.
  • Tests Maven/JUnit pour valider le modèle IR et la génération.
